# proc-002-split-section: セクション分割処理（1ステップ方式）

## 目的
HTMLファイルをCSVの目次情報に基づき、セクションごとのMarkdownファイルに分割する。
この処理には、画像ファイルのコピーとMarkdown内へのパス埋め込みも含まれる。

## 入力
- **HTMLファイル**: `_output/out-000-html/` 内の最大サイズの`.html`ファイル
- **CSVファイル**: `_output/out-001-outline.csv`
- **テンプレート**: `_template/proc-002-split-section.py`

## 出力
- **Markdownディレクトリ**: `_output/out-002-split-section/`
- **画像ディレクトリ**: `_output/images/`
- **ファイル形式**: Markdown (`.md`)
- **ファイル名**: CSVの`section_id`列をそのまま使用（例: `01_preface.md`, `02_chapter.md`）

## タスク

### 処理フロー
HTMLの構造を確認し、テンプレートスクリプトを調整して実行用スクリプトを作成・実行する。

**スクリプト出力先**: `_output/tmp/split_sections.py`

**処理内容**:
1.  HTMLファイルの自動検出（`images`を含む最大サイズのファイルを優先）。
2.  CSVファイルを読み込み、各セクションの情報を取得。
3.  HTMLからProject Gutenbergの定型文（ヘッダーやフッター）を除外。
4.  CSVの情報に基づき、各セクションのHTMLを抽出。
5.  HTMLからMarkdownへの変換処理：
    -   見出し、段落、太字、斜体などを変換。
    -   `<img>`タグを検出し、`src`属性から画像ファイルを特定。
    -   画像ファイルを`_output/images/`ディレクトリにコピー。
    -   キャプションを抽出し、`![キャプション](../images/画像ファイル名)`の形式でMarkdownを生成。
6.  変換後のMarkdownを`section_id.md`というファイル名で保存。

**実行コマンド**:
```
python _output/tmp/split_sections.py
```

### 結果報告
処理完了後、以下の情報を報告する：
- 生成したMarkdownファイル数
- コピーした画像ファイル数
- ファイルサイズの統計（最大、最小、平均）
- エラーがあったセクション（あれば）

## 1ステップ方式の利点
- **シンプル**: 処理が単一のスクリプトで完結するため、管理が容易。
- **確実性**: セクションのHTMLを解析する際に、その場で画像とキャプションを紐付けるため、情報の受け渡しミスがなく確実。
- **効率性**: HTMLのパースを一度しか行わないため、処理が高速。

## 見出しの処理
スクリプトは以下のパターンを自動的に`##`付きの見出しに変換する：
- `CHAPTER I.` → `## CHAPTER I.`
- `PREFACE` → `## PREFACE`

`h2`タグ内に画像と章タイトルが混在している場合も、画像とタイトルをそれぞれ適切に抽出し、Markdownに変換する。

## 重要事項
- **書籍ごとのHTML構造**: Project Gutenbergの書籍は、書籍ごとにHTMLの構造が微妙に異なる場合があるため、スクリプトが想定通りに動作しない場合は、HTML構造の確認とスクリプトの微調整が必要。
- **ファイル名**: 出力されるMarkdownのファイル名は、CSVの`section_id`と完全に一致させる。
- **パス参照**: Markdownファイルから画像ファイルへの参照は、相対パス（`../images/`）を使用する。

## トラブルシューティング
### 画像が表示されない場合
1.  `_output/images/`ディレクトリに、対象の画像ファイルがコピーされているか確認する。
2.  Markdownファイル内の画像パスが`![キャプション](../images/ファイル名)`の形式で正しく記述されているか確認する。
3.  元のHTMLファイル内で、画像(`<img>`)とキャプションの構造が、スクリプト(`proc-002-split-section.py`)内の正規表現が想定しているパターンと一致しているか確認する。特に、改行などが原因で正規表現のマッチが失敗していないか注意する。
