# proc-005-standardize: 翻訳統一方針書の作成（『1984』特別版）

## 1. 目的
`proc-003-system-prompt.md`（翻訳方針）と `proc-004-character.md`（キャラクター設定）を補完し、**『1984』特有の用語（ニュースピーク）や固有名詞の表記ルール**を厳格に定義する。
本作においては、用語の統一が「党による支配」を表現する重要な要素となるため、特に厳密な管理が求められる。

## 2. 本書の位置づけ
- **`proc-003`**: 作品全体の空気感と文体を定義。
- **`proc-004`**: キャラクターの個性と言語使用を定義。
- **本書 `proc-005`**: **用語、数字、表記の絶対的なルール**を定義。

---

## 3. タスク概要
原文Markdown群を解析し、以下の項目について網羅的なルールリストを作成する。

### 入力
- `_output/out-002-split-section/*.md`
- `_output/out-004-character.md`

### 出力
- `_output/out-005-standardize.md`

## 4. 詳細タスク

### ステップ1: ニュースピークおよび党用語の定義（最重要）
**目的**: 作品の世界観を決定づける造語の訳語を統一する。

1.  **用語の抽出**:
    - 原文から大文字で始まる用語、短縮語（Ingsoc, Minitrue, doublethink, thoughtcrime, telescreen, Big Brotherなど）を抽出。
2.  **訳語の決定**:
    - 既存の優れた翻訳（新訳含む）を参考に、現代の読者に伝わりやすく、かつ不気味さを感じる訳語を選定する。
    - **ルール例**:
        - `Big Brother` → `ビッグ・ブラザー`（偉大なる兄弟、とはしない）
        - `Ingsoc` → `イングソック`（党のイデオロギー）
        - `Minitrue` → `真理省`（略称として定着させる）
        - `Miniluv` → `愛情省`
        - `Thought Police` → `思考警察`
        - `doublethink` → `二重思考`
        - `facecrime` → `顔面犯罪`
    - **出力**: `| 原文 | 日本語統一表記 | 解説・備考 |` のテーブルを作成。

### ステップ2: 固有名詞データベースの構築
1.  **人名の統一**: `_output/out-004-character.md` と整合性を取る。
2.  **地名・場所**: `Airstrip One`（エアストリップ・ワン / 第一空挺地域）、`Victory Mansions`（勝利マンション / ビクトリー・マンション）などの統一。

### ステップ3: スローガンと歌の翻訳ルール
1.  **党のスローガン**:
    - `WAR IS PEACE`
    - `FREEDOM IS SLAVERY`
    - `IGNORANCE IS STRENGTH`
    - これらをどう訳し（「戦争は平和なり」か「戦争は平和だ」か）、どう表記するか（太字、鍵括弧など）のルールを定める。
2.  **歌（「オレンジとレモン」など）**:
    - 劇中に登場するわらべ歌の訳詞方針を定める（既存の訳を尊重するか、リズムを優先して新訳するか）。

### ステップ4: 記号・見出し・数字のルール（縦書き対応）
`sherlock-kdp-short-01`の基本ルールを踏襲しつつ、**縦書き出版**に合わせて厳格化する。

1.  **段落の字下げ禁止**:
    -   Markdown形式で出力するため、段落の冒頭に全角スペースや空白を入れてはならない。

2.  **見出し**: `## 第一部`、`### 第一章` の形式に統一（漢数字）。
3.  **数字**:
    - **原則**: **すべて漢数字**。
    - **年号・日付**: `1984年` → `一九八四年`、`4月4日` → `四月四日`。
    - **時間**: `13時` → `十三時`。
    - **数式**: `2+2=5` → `二＋二＝五`。
    - **例外なし**: アラビア数字は使用しない。
4.  **アルファベット**:
    - **1文字**: **全角**に統一（例：`plan B` → `プランＢ`）。
    - **略語**: `B.B.` → `Ｂ・Ｂ`。
    - **単語**: 極力カタカナ化するが、残す場合は全角を検討する。

### ステップ5: 文末表現と「だ・である」調の徹底
1.  **地の文**: 原則として「だ・である」調。ハードボイルドかつ冷徹なトーンを維持。
2.  **思考描写**: ウィンストンの内面描写において、感情が高ぶる箇所での文体の揺らぎ（体言止め、短文の連打）を許容・推奨するルールを記述。

### ステップ6: 最終成果物の生成
- 上記情報を統合し、`_output/out-005-standardize.md` を生成する。
