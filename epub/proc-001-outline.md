# proc-001-outline: 目次抽出処理（自動修正機能付き）

## 目的
HTMLファイルから目次情報を抽出し、CSVファイルとして出力する。
処理結果が期待と異なる場合、スクリプトを自動的に修正し、再実行する。

## 入力
- **HTMLファイル**: `_output/out-000-html/` 内の `*-images.html` ファイル

## 出力
- **CSVファイル**: `_output/out-001-outline.csv`
- **フォーマット**: `type,title,section_id`

## 期待される結果の定義
処理の成否を判断するための具体的な検証項目を定義する。

- **検証項目1: フォーマット**:
  - 出力は `type,title,section_id` の3列CSV形式であること。
- **検証項目2: 項目数**:
  - 今回の書籍『高慢と偏見』の場合、目次項目が62個であること。
- **検証項目3: 内容**:
  - `type` 列に `preface` と `chapter` が含まれていること。
  - `title` 列が空でないこと。
  - `section_id` 列が空でないこと。
- **検証項目4: 表記の統一**:
  - `title` 列のすべての値が **大文字** であること。（例: `PREFACE`, `CHAPTER I`）

## タスクフロー

1.  **目次抽出スクリプトの実行**:
    ```bash
    chmod +x _template/proc-001-extract-toc.py
    _template/proc-001-extract-toc.py
    ```

2.  **結果の検証**:
    - `_output/out-001-outline.csv` ファイルが生成されたか確認する。
    - ファイルの内容が上記の「期待される結果の定義」をすべて満たしているか検証する。

3.  **自動修正 (検証に失敗した場合)**:
    - **原因分析**:
        1. `_template/proc-001-extract-toc.py` のソースコードを解析する。
        2. `_output/out-000-html/*-images.html` のHTML構造を解析する。
        3. なぜ期待通りの結果にならなかったのか（例: HTMLの目次ブロックのCSSセレクタが異なる、など）を特定する。
    - **スクリプト修正**:
        - 原因を解決するために、Pythonスクリプトのロジックを修正する。
    - **再実行と再検証**:
        - 修正したスクリプトを再実行し、結果が「期待される結果の定義」を満たすまでこのサイクルを繰り返す。

## スクリプトの基本処理内容
- 入力HTMLファイル内の目次ブロックを検索する。（初期セレクタ: `<p class="toc">`）
- ブロック内のすべての `<a>` タグを解析する。
- `href` 属性を元に、`type`, `title`, `section_id` を生成する。
- 結果を `_output/out-001-outline.csv` に上書き保存する。
