# プロセス指示書: 分冊版ソース準備 (082)

## 概要
手動編集済みのマスターデータ（`out-065`）から、分冊出版（シングルカット）用のディレクトリ構造を作成し、必要なファイルを配置します。
出力先は `_output/out-080-single/chap[章番号]/source/` とします。

## 前提条件
- `epub/proc-070-assemble.md` が完了しており、`_output/out-061-assemble/` にマスター原稿が存在すること。

## 手順

### 1. 対象章の選択
ユーザーから、分冊化作業を行う章番号（例: `01`）を指定させます。

### 2. ディレクトリ作成
以下のディレクトリ構成を作成します。
`_output/out-080-single/chap[章番号]/source/`

例:
```bash
mkdir -p _output/out-080-single/chap01/source
```

### 3. ファイルの配置
以下のルールに従ってファイルを `source` ディレクトリにコピー・生成します。
ファイル名は、結合時（Pandoc処理時）のソート順序 `00` -> `01` -> `99` が崩れないように維持します。

#### A. 本文ファイル
- マスターの `[章番号]_chap[章番号].md` をコピーします。
- 出力ファイル名: `01_chap[章番号].md`

#### B. 共通ファイル（はじめに・おわりに・登場人物）
以下のファイルをコピーした後、**その章（エピソード）の内容に合わせてAIが内容を自動修正（リライト）します**。

1.  **登場人物 (`00_02_characters.md`)**
    - **処理**: ファイルをコピーした後、対象の章（例: `01_chap01.md`）の本文を分析します。
    - **編集指示**:
        - その章に**登場しないキャラクター**の項目を削除してください。
        - 「シャーロック・ホームズ」「ジョン・H・ワトソン」などのレギュラーメンバーは残します。
        - 見出しを「登場人物紹介（ボヘミアの醜聞）」のように調整してください。

2.  **はじめに (`00_01_foreword.md`)**
    - **処理**: ファイルをコピーした後、内容をリライトします。
    - **編集指示**:
        - 「本書は全集である」といった記述を、「本書は『シャーロック・ホームズの冒険』より第[章番号]章『[章タイトル]』を収録した**分冊版**である」という文脈に書き換えてください。
        - 新訳の特徴（読みやすさ等）に関するアピールは残してください。

3.  **おわりに (`99_01_afterword.md`)**
    - **処理**: ファイルをコピーした後、内容をリライトします。
    - **編集指示**:
        - 読者への感謝の言葉を、短編1話を読み終えた読者に向けたものに調整してください。
        - 次の巻や、合本版への興味を誘うような一文（「他の冒険もぜひお楽しみください」など）を追加してください。
        - **重要**: Amazon（Kindle）でのレビューや星評価をお願いする一文を必ず追加してください。「星評価だけでも励みになる」といった謙虚かつ前向きな表現にしてください。

#### C. 奥付（分冊用）の生成
- マスターの `metadata.yaml` を読み込みますが、タイトルを「分冊版」用に書き換えて、新規に `99_02_colophon.md` を作成します。
- **タイトル**: `『シャーロック・ホームズの冒険』より「[章タイトル]」`
- **発行日**: 本日の日付

### 4. 確認
作成されたディレクトリ `_output/out-080-single/chap[章番号]/source/` の内容をリストアップし、ユーザーに編集を促します。
