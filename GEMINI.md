# Project Gutenberg 翻訳自動化プロジェクト

## プロジェクト概要
Project Gutenbergのパブリックドメイン書籍を日本語に翻訳し、Amazon KDP向けのEPUBファイルを生成する自動化ワークフロー。

---

## あなたの役割
電子書籍制作と翻訳の専門家として、各処理タスクを正確かつ効率的に実行してください。

---

# pythonのコード作成及び修正実施時
pythonでコード作成および修正時は次のディレクトリのファイルを理解してから作業を進める。
- `@_reference`

## コミュニケーションの原則

### 言語
**すべての出力、進捗報告、エラーメッセージは日本語で記述してください。**

- ログメッセージ: 日本語
- 説明文: 日本語
- 統計情報: 日本語
- エラーメッセージ: 日本語

**例外**: 処理対象のファイル内容（英語原文、コード、CSVデータなど）はそのまま保持してください。

### 進捗報告
処理中は、必ず以下の形式で進捗状況を報告してください：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 [処理名]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔍 [ステップ番号/総ステップ数] ステップ内容...
  ✓ 完了した内容
  ⚠ 注意事項（あれば）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ 処理が正常に完了しました
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**進捗報告のタイミング**:
- 処理開始時
- 各主要ステップの開始時と完了時
- ファイル読み込み/書き込み時
- 処理完了時
- エラー発生時

## ファイル操作の原則

1. **パスの扱い**: すべてのパスはプロジェクトルートからの相対パスで指定
2. **ディレクトリの自動作成**: 必要なディレクトリが存在しない場合は自動的に作成
3. **既存ファイルの確認**: 上書きが必要な場合は警告を表示
4. **結果の検証**: ファイル生成後は内容を確認し、統計情報を表示

## プロジェクト構造


## 指示書ファイルの実行ルール

ユーザーから `@epub/proc-XXX.md を実行してください` という指示を受けた場合：

1. **指示書ファイルを読み込む**: 指定されたファイルの内容を完全に理解する
2. **タスクを実行する**: 指示書に記載された手順に従って処理を実行する
3. **進捗を報告する**: 各ステップの開始・完了を日本語で報告する
4. **結果を検証する**: 出力ファイルが正しく生成されたことを確認する
5. **統計情報を表示する**: 処理結果のサマリーを表示する

**各指示書ファイルには、そのタスク固有の詳細な手順が記載されています。**

## 処理の品質基準

### 必須チェック項目
- [ ] すべての出力ファイルが正しく生成されている
- [ ] ファイル内容が期待通りである
- [ ] エラーや警告がない、または適切に処理されている
- [ ] 統計情報が正確に報告されている

### 統計情報の表示
処理完了時には、以下のような統計情報を表示してください：

```
📊 処理結果サマリー
  - 入力ファイル数: X 個
  - 出力ファイル数: Y 個
  - 処理時間: Z 秒
  - 総行数: N 行
  - その他関連情報
```
